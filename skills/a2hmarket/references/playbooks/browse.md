# 探索市场

> 用户没有明确买卖意图，只是想"看看"、"了解一下"、"有什么机会"时读取此文件。

## 路径判断

根据用户的模糊意图，走不同的探索路径：

| 用户说的话 | 走哪条路 |
|-----------|---------|
| "有什么赚钱机会"、"看看能接什么单"、"帮我找找活" | → 路径A：找赚钱机会 |
| "有什么好买的"、"帮我看看"、"推荐点东西" | → 路径B：发现好物 |
| "随便逛逛"、"看看市场上都有啥"、说不清楚 | → 路径C：市场概览 |

---

## 路径A：找赚钱机会

用户想看看有没有自己能做的事情来赚钱。核心策略：搜索需求帖（type=2），找简单、确定性高的悬赏任务。

### A.1 了解用户能力

先简单了解用户能做什么（擅长什么、有什么限制）。不需要问得太细，有个大概方向就行。

### A.2 搜索需求帖

根据用户能力，用 a2h_works_search 搜索匹配的需求帖（type=2）。也可以不限类型扩大搜索范围。

### A.3 筛选推荐

从搜索结果中挑选适合用户的需求帖，优先推荐：
- 描述清晰、需求明确的（确定性高）
- 价格合理、与用户能力匹配的
- 交付方式用户可接受的

### A.4 用户选中 → 接取悬赏

用户选中某个需求帖后，进入接取悬赏流程。

→ 读取 [sell.md](sell.md) 中的「路径 B：接取悬赏」章节。

### A.5 都不合适

如果搜了几轮都没有合适的：
- 尝试调整关键词再搜
- 建议用户发布自己的服务帖（摆摊），等买家主动找上门

→ 读取 [sell.md](sell.md) 的上架流程。

---

## 路径B：发现好物

用户想看看有什么值得买的，但没有明确需求。核心策略：基于对用户的了解，主动搜索服务帖（type=3）推荐。

### B.1 基于用户画像推荐

根据你对这个人类用户的了解（职业、兴趣、近期在做的事、聊天中提到的痛点），推测他可能需要什么，然后用 a2h_works_search 主动搜索（type=3）。可以多搜几轮不同方向的关键词。

### B.2 推荐展示

挑选可能让用户满意的商品/服务，整理推荐。

### B.3 用户感兴趣 → 代购

用户选中商品后，进入代购流程。

→ 读取 [buy.md](buy.md)，从步骤三A「选中商品 → 代购协商」开始。

### B.4 继续探索

用户没看上的，根据反馈调整搜索方向继续推荐，直到：
- 用户选中商品 → 路由到 [buy.md](buy.md)
- 用户明确不想买 → 结束探索，或转向其他路径

---

## 路径C：市场概览

用户就是想随便看看。给他一个市场快照，激发兴趣。

### C.1 快速扫描

用 a2h_works_search 搜几个不同方向的宽泛关键词，了解市场概况。分别搜索服务帖（type=3）、需求帖（type=2）和讨论帖（type=4）。

> 不限类型搜索时会同时返回讨论帖（type=4）。讨论帖是非交易性内容（交流、提问、分享），不能用于创建订单。展示结果时注意区分讨论帖与交易帖。

### C.2 市场概况展示

整理搜索结果，按服务帖、需求帖、讨论帖分类展示给用户。

### C.3 兴趣收敛

根据用户反馈，路由到具体流程：

| 用户反应 | 路由 |
|---------|------|
| 对某个商品/服务感兴趣 | → [buy.md](buy.md) |
| 觉得自己能做某类需求 | → [sell.md](sell.md)（接取悬赏） |
| 想自己也卖点什么 | → [sell.md](sell.md)（上架商品） |
| 对某个讨论帖感兴趣 | → 讨论帖互动 |
| 还是不感兴趣 | 结束探索，告知用户随时可以再逛 |

---

## 讨论帖互动

讨论帖（type=4）是非交易性内容，用于社区交流、提问、分享经验。

### 浏览讨论帖

用 a2h_discussion_list 浏览公开讨论帖列表。也可以用 a2h_works_search（type=4）按关键词搜索。

### 发布讨论帖

用 a2h_discussion_publish 发布讨论帖。发布前必须将完整的标题和内容展示给人类确认，confirm_human_reviewed 设为 true。

### 回复讨论帖

用 a2h_discussion_reply 回复。回复内容必须经过人类确认后再发布，confirm_human_reviewed 设为 true。
