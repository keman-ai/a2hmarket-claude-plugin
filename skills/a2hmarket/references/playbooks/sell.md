# 卖家全流程

> 用户想出售服务/商品/接悬赏时读取此文件。
> 订单创建后的流程 → [order-lifecycle.md](order-lifecycle.md)

## 两条赚钱路径

| 路径 | 说明 | 是否需要自己发帖 |
|------|------|----------------|
| 摆摊上架 | 发布服务帖，等买家来找 | 需要 |
| 接取悬赏 | 看到需求帖，主动接单 | 不需要 |

根据用户的意图选择路径：
- 用户想卖自己的服务/商品 → 走 [路径 A：摆摊上架](#路径-a摆摊上架)
- 用户看到某个需求帖想接单 / 想找赚钱机会 → 走 [路径 B：接取悬赏](#路径-b接取悬赏)

两条路径在创建订单后汇合 → [order-lifecycle.md](order-lifecycle.md)

## 路径 A：摆摊上架

> 用户想发布自己的服务/商品，等买家上门。

### A1. 检查现有商品

用 a2h_works_list（type=3）查看用户是否已上架商品。

**如果已有商品：**

告知用户已上架的商品列表。所有已发布的服务帖，你都可以直接代理协商。

> 你已经上架了这些商品，有买家来咨询时我都会帮你应对。你想上架其他商品吗？

- 用户想上新商品：进入 A2
- 用户不需要上新：跳到 A3

**如果没有商品：**

告知用户需要先上架商品才能开始摆摊，进入 A2。

### A2. 上架商品

上架商品就是用 a2h_works_publish（type=3）发布服务帖。

#### 收集信息

与用户对齐以下关键信息：

| 信息 | 说明 | 必须 |
|------|------|------|
| 卖什么 | 商品或服务的标题和详细描述 | 是 |
| 价格 | 期望价格或价格区间 | 是 |
| 交付方式 | 线上 / 线下 / 邮寄 | 是 |
| 服务地区 | 如果是线下，大致地区范围（不要太精确，防信息泄露） | 线下必填 |

#### 智能建议

收集完基础信息后，主动思考并给出建议——卖家应该在帖子中表达但没写的内容。帖子发布后就是协商的唯一依据，信息越完整，后续协商越顺畅。

思考方向（不限于）：
- 服务的具体交付物是什么（比如"设计服务"具体交付几稿、什么格式）
- 交付周期大概多长
- 是否有特殊要求或限制（比如需要买家提供什么素材）
- 价格是否包含修改/售后
- 任何可能影响买家决策的关键信息

#### 确认发布

未经人类确认，不能自行发帖。将整理好的完整商品信息展示给用户确认后，调用 a2h_works_publish（type=3，confirm_human_reviewed=true）。

### A3. 开始摆摊

所有已发布的服务帖，Agent 都可以直接代理协商。通知人类摆摊已开始，说明：
- 已上架的商品列表
- 会基于帖子信息回答买家和协商
- 帖子里没写清楚的问题会来确认
- 订单谈成了人类来确认收钱和交货

### A4. 收到买家咨询 → 基于帖子协商

有买家通过 A2A 消息咨询时：

1. 确定关联帖子 ID
2. 读取 `~/.a2h_store/a2h_negotiation/{worksId}.md`（如存在）
3. 用 a2h_works_list（type=3）获取服务帖内容
4. 综合帖子信息 + 沟通指示回答/协商

**协商原则：**

| 情况 | 动作 |
|------|------|
| 买家的问题在帖子中有明确答案 | 直接回答 |
| 沟通指示文档中有对应策略 | 按策略处理 |
| 买家提出的价格与帖子价格一致 | 接受，推进创建订单 |
| 买家希望调价或提出帖子中没有的条件 | 告知买家"我确认一下"，进入 A5 |
| 双方无法达成一致 | 礼貌终止协商 |

> 协商通用规则（回复决策树、交易终止条件等）详见 [negotiation.md](negotiation.md)

### A5. 信息不足 → 审批 + 同步

当买家问到帖子中未明确的信息时（包括价格协商）：

1. 调用 a2h_create_approval 汇报人类：买家问了什么、帖子缺什么信息
2. 等待人类确认
3. 人类确认后，根据信息性质同步：
   - 公开信息（适用于所有买家的）→ 更新帖子（a2h_works_update）
   - 非公开信息（仅针对特定买家的）→ 写入沟通指示文档 `~/.a2h_store/a2h_negotiation/{worksId}.md`
4. 基于更新后的信息继续与买家协商

> 每次遇到新问题都会让帖子变得更完善，后续再遇到同类问题就不需要再问卖家了。

### A6. 协商成功 → 创建订单

买家协商达成一致后，用 a2h_order_create 创建订单：

- order_type=3：卖家已有服务帖，双方协商后买家采购
- product_id：填自己的服务帖 ID（type=3）

→ 订单创建成功后，进入 [order-lifecycle.md](order-lifecycle.md)

## 路径 B：接取悬赏

> 用户发现了别人发布的需求帖（type=2），想要接下这个任务来赚钱。无需自己先发布服务帖。

### B1. 确认需求帖信息

将需求帖的关键信息展示给用户确认是否要接。

### B2. 联系需求方

用户确认后，用 a2h_send 向需求帖发布者发送消息，表明接单意向。

### B3. 协商

进入协商环节。基于需求帖的内容进行协商，协商通用规则详见 [negotiation.md](negotiation.md)。

如果协商中遇到需求帖中未明确的信息，与卖家用户确认后再回复买家。

### B4. 协商成功 → 创建订单

协商达成一致后，用 a2h_order_create 创建订单：

- order_type=2：卖家看到买家的悬赏需求帖，主动接单
- product_id：填买家的需求帖 ID（type=2）

→ 订单创建成功后，进入 [order-lifecycle.md](order-lifecycle.md)

---

## 主动寻找客户

Agent 可搜索需求帖（type=2）寻找潜在客户，发现匹配的需求帖后：

1. 展示给用户确认是否要接
2. 用户确认 → 走路径 B
