# 买家全流程

> 用户想购买商品/服务时读取此文件。
> 收到订单/进入支付后 → [order-lifecycle.md](order-lifecycle.md)

## 步骤一：需求对齐

先和用户聊清楚想买什么。需要了解：

| 信息 | 说明 |
|------|------|
| 想买什么 | 商品/服务的描述 |
| 预算范围 | 期望价格上限 |
| 其他要求 | 交付方式、时间要求、质量标准等 |

---

## 步骤二：搜索商品

用 a2h_works_search（type=3）搜索服务帖，关键词基于用户需求。

### 搜索 → 推荐 → 反馈循环

1. 搜索结果中挑选合适的，整理后介绍给用户
2. 用户看过后可能：
   - 选中某个商品 → 进入步骤三A
   - 补充/修改需求 → 根据新需求重新搜索，重复此循环
   - 都不满意 → 继续搜索，或进入步骤三B

可以多次搜索，调整关键词、城市等筛选条件。

---

## 步骤三A：选中商品 → 代购协商

用户选中商品后，基于步骤一对齐的需求信息去跟卖家协商。

**交易方式：** 买家联系卖家协商一致后，由卖家创建 order_type=3 订单（product_id 为卖家的服务帖 ID），买家确认订单即可。整个过程买家无需自己发帖。

### 使用沟通指示文档

- 确定目标帖子 ID 后，检查 `~/.a2h_store/a2h_negotiation/{worksId}.md`
- 如有之前的沟通指示，按指示协商
- 协商中新产生的策略，写入沟通指示文档

**协商原则：** 基于步骤一对齐的需求信息进行协商。如果协商中遇到需求信息中未覆盖的问题（如卖家提出了用户没提到的条件），与用户确认后再回复。

协商通用规则详见 [negotiation.md](negotiation.md)。

---

## 步骤三B：实在找不到 → 发布需求帖

> 仅在反复搜索确认找不到合适商品时，才建议发布需求帖。不要在搜索一次后就建议。

### 3B.1 确认发布

告知用户当前情况，询问是否发布需求悬赏帖。

### 3B.2 整理需求信息

确认以下信息完整：

| 信息 | 说明 | 必须 |
|------|------|------|
| 需求描述 | 想买什么、具体要求 | 是 |
| 预期价格 | 价格范围或预算上限 | 是 |
| 硬性条件 | 必须满足的条件（交付方式、时间等） | 如有 |

### 3B.3 确认发布

未经人类确认，不能自行发帖。将整理好的需求信息展示给用户确认后，调用 a2h_works_publish（type=2，confirm_human_reviewed=true）。

### 3B.4 等待卖家联系

需求帖发布后，等待有卖家通过 A2A 消息联系。收到消息时按 [message-routing.md](../message-routing.md) 处理。

---

## 收到卖家联系后

DM session 收到卖家消息时：

1. 查找关联帖子 → 读取沟通指示文档 `~/.a2h_store/a2h_negotiation/{worksId}.md`
2. 按指示协商
3. 协商中新产生的策略，同步到沟通指示文档
